# 计划：代码查看器布局与交互修复（v0.3.1 patch）

> 来源：用户截图反馈（TextViewer 长行换行侵占行号栏）+ 多条交互诉求。
> 用户要求"跟一般软件（VS Code 等）操作逻辑一样"。
> 本批是已有功能的 bug 修复 + 交互对齐 → 按附录 E 是 **PATCH** → 版本号 **0.3.1**。

---

## 问题清单

### 1. 行号栏与代码栏布局错误（核心，截图反映的问题；TextViewer + DiffViewer 同样适用）
- **现状**：`src/renderer/styles/global.css:2240` `.file-text-viewer` / `.file-text-line` 用 `white-space: pre-wrap; word-break: break-all`；行号是 `inline-block` span 放行首。长行换行时，换行文字从容器的 `x=0` 开始（在 inline-block 下方），**盖住行号栏**。DiffViewer 当前根本无行号槽（v0.3.2 时刻意没加）。
- **用户期望（正确，VS Code 同款）**：
  - 行号栏与代码栏是**两个独立列**（grid），不是"每行 inline 序列"
  - 行号栏固定宽度，代码栏独立
  - 代码**不换行**（`white-space: pre`），超出部分水平滚动
  - 水平滚动时行号栏**保持固定**（sticky left:0），只有代码横向滚动

### 2. 水平滚动
- **现状**：无（pre-wrap 永不超出）。
- **期望**：代码宽于视口 → 出现水平滚动条；行号栏 sticky 不动。

### 3. 中键拖动 pan（auto-scroll / 手型工具）
- **现状**：不支持。
- **期望**：按住鼠标中键拖动 → 上下 + 左右自动滚动（浏览器 / VS Code / Acrobat 同款）。方向跟随鼠标偏移。

### 4. Diff 语法高亮（需确认，可能误解）
- **用户观察**："一般代码有语法高亮，但 diff 版本文件没有？"
- **代码层验证（已查）**：DiffViewer 自 v0.3.0 起**有** hljs 双层高亮，CSS 规则 `.diff-line-body .hljs-*` 在（global.css:2447-2469），`DiffViewer.tsx:130` 调 `highlightLine()` 正常。
- **最可能原因**：用户看的 diff 文件扩展名**不在 `EXT_TO_HLJS` 表**（如 .go/.rs/.java/.rb），回退到纯 diff 语言（只有行底色、无 token 着色）→ 视觉上像"没高亮"。这是**设计如此**（按需 11 语言），不是 regression。
- **行动**：向用户澄清 + 视需求扩展语言表（低成本，加 import 即可）。

## 实现计划

### 阶段 1：布局重构（TextViewer + DiffViewer）
**CSS 改 `src/renderer/styles/global.css`**：
- 容器：`display: grid; grid-template-columns: auto 1fr; overflow: auto`（双向滚动容器）。
- 推荐结构（每逻辑行）：
  ```
  <div class="file-text-line">  ← grid row: [行号 cell][代码 cell]
    <span class="file-line-number"> ← position: sticky; left:0; z-index:1; bg=行底色
    <span class="file-text-line-content"> ← white-space: pre (不换行)
  </div>
  ```
- 删 `white-space: pre-wrap` / `word-break: break-all`，改 `white-space: pre`。
- 行号 cell：`position: sticky; left: 0; background: var(行底色); z-index: 1`（水平滚动时钉住）。

#### TextViewer
- 现状：已有行号 span（v0.3.2 加的）。只改 CSS（inline-block → grid cell + sticky + pre）。

#### DiffViewer（本批重点补齐，与 TextViewer 对等）
- **现状**：DiffViewer **没有行号槽**（v0.3.2 时刻意没加，理由是“old/new 两套行号语义复杂”）。只有 `sign`（+/-）+ `body`。
- **本批决策**：diff 也加行号槽，布局与 TextViewer 同构。行号语义：**按 unified diff 的文件行号**（从 hunk header `@@ -a,b +c,d @@` 解析 new-side 行号；del 行用 old-side，add/ctx 用 new-side）。hunk 间的空区不编行号。实现上 buildRows 时跟踪行号计数器。
- 三栏布局：`[行号 sticky][sign][body white-space:pre]`。sign 与行号都可 sticky left（连着钉住），或行号 sticky + sign 随 body 滚（视觉上 sign 紧贴行号更自然，两者都 sticky）。
- **行号背景跟随行底色**：diff 各行底色不同（add 绿/del 红/hunk 蓝/ctx 透明），sticky 行号的 background 必须跟随 `.diff-line-${kind}` 的底色，否则水平滚动时代码会透过行号看到（行号透明的话）。
- **DOM 要改**（不是“基本不变”）：DiffViewer.tsx 每行加一个行号 span（在 sign 之前），buildRows 产出 lineNum 字段。

**组件改 `TextViewer.tsx` / `DiffViewer.tsx`**：
- TextViewer：DOM 基本不变（已有行号 span），主要是 CSS。
- DiffViewer：**DOM 要改**——每行加行号 span，buildRows 解析 hunk 算行号（见上 DiffViewer 小节）。

### 阶段 2：中键 pan（新 hook）
**新文件 `src/renderer/hooks/useMiddleClickPan.ts`**：
```
useMiddleClickPan(containerRef):
  mousedown(button===1) → 记录起点 {x,y,scrollLeft,scrollTop}，设 cursor: grabbing，
                          preventDefault（阻止浏览器原生中键自动滚动光标）
  mousemove(panning)    → container.scrollLeft = start.scrollLeft - (e.clientX - start.x)
                          container.scrollTop  = start.scrollTop  - (e.clientY - start.y)
  mouseup / mouseleave / blur → 停止，恢复 cursor
```
TextViewer / DiffViewer / MarkdownViewer 容器接入。

### 阶段 3：澄清 + 视需扩展 diff 语言表
- 向用户说明 diff 高亮现状 + EXT_TO_HLJS 覆盖的 11 语言（ts/js/py/json/bash/yaml/markdown/xml/c/cpp/cs）。
- 若用户常用语言不在（go/rust/java/ruby/php 等）→ `highlight.ts` 加 import + EXT_TO_HLJS 映射（每个 ~3-8KB）。

### 阶段 4：验证 + 文档
- typecheck / lint / lint:css / test / build 全绿
- 手动测：① 长行不换行 ② 水平滚动 ③ 行号 sticky 不动 ④ 中键 pan 上下左右 ⑤ diff 高亮（用 .ts 文件测）
