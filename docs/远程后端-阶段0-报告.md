## 3. ipc-protocol 改动(`docs/ipc-protocol.md`)— **v2.0 major bump**

`PROTOCOL_VERSION` 1 → 2(字段重命名 `windowId`→`clientId` 触发,见 §10.2)。

- **§2.3 命令信封**:`windowId` → `clientId`(本地 in-process client = windowId,远程 WS client = 握手分配的稳定 id)。
- **§2.4 事件信封**:加 `targetClientId`(定向事件)。
- **§2.6 Transport 抽象(新增)**:Transport-Local(Electron IPC,零改动)+ Transport-Ws(WebSocket+TLS),dispatcher 设计,两者跑同一套语义。
- **§4 Handshake**:加 WS 握手分支(TLS 指纹确认 + token 认证 + clientId 分配/复用)。
- **§5.2 Session**:`cmd:session:claim` 复用扩展(client 接管);新增 `cmd:session:release`;**断线自动 release**(WS 关闭/心跳超时 → 该 client 持有的 session owner 置空,session 不销毁)。
- **§10.1**:版本 bump + 顶部 v2.0 changelog。

语义层(§3 / §5-§9 命令事件清单 / 错误码 / 时序)**不动**。
