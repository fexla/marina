## 真问题修复清单

### #1 · 数字框 spinner 箭头不主题化

- **现象**:`<input type=number>` 右侧上下小箭头默认深灰,在 Rose Pine Dawn 等浅色主题下露白
- **根因**:Chromium 的 `::-webkit-inner-spin-button` 颜色几乎不可主题化,`accent-color` 也不覆盖箭头本身
- **修复**:`global.css` 直接 `appearance: none` 隐藏 spinner;用户键盘 / 粘贴输入数值
- **文件**:`src/renderer/styles/global.css`

### #2 · UI 字体设置不生效

- **现象**:设置页改 "UI 字体" 后无视觉变化
- **根因**:`uiFontFamily` 写入 `settings.json`,但**没有任何代码**把它注入到 DOM。`global.css` 的 `--ui-font-family` 是硬编码 `:root`
- **修复**:`App.tsx` 加 `useEffect`,settings 变化时 `document.documentElement.style.setProperty('--ui-font-family', uiFontFamily)`。`terminalFontFamily` 顺手一起接通
- **文件**:`src/renderer/App.tsx`

### #3 · 新建 shell 闪一下控制台窗口

- **现象**:从 + 按钮或模板按钮起 session 时,主屏中央可见 conhost 黑窗一闪
- **根因**:`node-pty` 默认 ConPTY,但 Win 10 1809 之前的版本或某些组策略下会 fallback 到 winpty(winpty 显式起 conhost.exe 因此闪窗)
- **修复**:`session-manager.ts` spawn 选项显式 `useConpty: true`,强制不走 winpty fallback
- **文件**:`src/main/session-manager.ts`(并扩展 `PtySpawnFn` 类型)

### #4 · 复选框/单选粗糙

- **现象**:原生方形 checkbox + 圆形 radio + `accent-color: var(--iris)` 在 Windows 上深色主题下背景与勾选色对比度不够,而且边框看上去廉价
- **修复**:`global.css` 自绘
  - checkbox:16×16 框 + `:checked::after` 旋转 45° 双边白勾
  - radio:16×16 圆 + `:checked::after` 居中圆点
  - 边框 hover → iris;选中 → iris 实心
- **文件**:`src/renderer/styles/global.css`

### #5 · 拖拽遮罩偶尔无法消失

- **现象**:从 Explorer 往 Marina 拖文件夹中途松手到外部,半透明 "松开鼠标 — 在该文件夹打开新终端" 遮罩卡住不消失
- **根因**:`MainPane.handleDragLeave` 只在 `e.currentTarget === e.target` 时清状态。拖出窗口外松手时本窗口没有 dragleave 事件,状态卡 `true`
- **修复**:`MainPane.tsx` 加 `window` 级 `dragend` / `drop` 监听兜底,无论事件源都强清 `dragOver=false`
- **文件**:`src/renderer/components/MainPane.tsx`

### #6 · Git Bash 路径追踪没生效

- **现象**:Git Bash session 内 `cd /c/foo` 后 tab 上不出 ⚠️,Marina 看不到 cwd 变化
- **根因(双层 bug)**:
  1. **packed 模式下 hook 文件根本不在 asar**!`electron-builder.yml` 的 `files: out/**` 只打包 `out/`,`src/shell-hooks/` 不在。pwsh / cmd / bash 三种 shell 都受影响,但 cmd 用 PROMPT env 不依赖外部脚本所以撑住了
  2. 即使加载,bash.sh 用 `$PWD` 在 Git Bash 上返回 Unix 风格(`/c/Users/foo`),Marina PathManager 按 Windows 风格归类,无法匹配
- **修复**:
  1. `electron-builder.yml` 加 `extraResources: src/shell-hooks → shell-hooks`,把 hook 脚本放 `<install>\resources\shell-hooks\` 与 `app.asar` 同级
  2. `session-manager.ts` 的 `defaultHookFileResolver` 检测 `__dirname.includes('app.asar')` 切换到 `process.resourcesPath/shell-hooks`,dev 模式仍走源码目录
  3. `bash.sh` 探测 `cygpath` 存在则 `cygpath -w "$PWD"` 转 Windows 风格再发 OSC 1337,POSIX 系统(无 cygpath)直接发 `$PWD`
- **文件**:`electron-builder.yml` + `src/main/session-manager.ts` + `src/shell-hooks/bash.sh`

### #7 · Sidebar session 右键缺 2 项

- **现象**:Sidebar session 右键只 5 项(含 hint),与 Tab 右键不一致;spec 6.3 要求 6 actionable
- **修复**:补 "复制路径" + "在 Explorer 中显示" 到 Sidebar session 菜单,与 Tab 对齐
- **文件**:`src/renderer/components/Sidebar.tsx`

### #8 · 重命名菜单始终灰显

- **现象**:Tab 右键 + Sidebar session 右键的 "重命名" 始终灰显,即使本窗口持有该 session
- **根因**:`disabled: !isMine`(或 `variant !== 'mine'`)把 **orphan**(无主)session 也灰掉了。spec 6.3 + milestone-1 §2.3 描述是 "不是本窗口持有时" 灰显,orphan 不属于"他人持有"。再加上 SessionManager.renameSession IPC 根本不校验 owner,实质允许 orphan 改名
- **修复**:统一改为 `disabled: variant === 'other'`(Tab)/ `disabled: ownedByOther`(Sidebar),orphan + 本窗持有 都允许
- **文件**:`src/renderer/components/Sidebar.tsx` + `src/renderer/components/MainPane.tsx`

### #9 · 用托盘图标作 app/任务栏图标

- **现象**:exe 内嵌的还是 Electron 默认灰齿轮;任务栏 / 桌面快捷方式 / Explorer 右键菜单图标都是默认
- **修复**:
  - 写 `scripts/generate-icon.cjs` — 纯 JS PNG 编码器(0 deps)+ ICO 容器,256×256 渲染与 `build/icon.svg` 同设计(Rose Pine 紫底 + Iris ">_" + Gold 光标)
  - 产物 `build/icon.png` + `build/icon.ico` 入仓
  - `electron-builder.yml` 三处 icon 行打开:`win.icon` / `nsis.installerIcon` / `nsis.uninstallerIcon` 都指向 ICO
  - `package.json` 加 `npm run generate-icon` script
- **文件**:`scripts/generate-icon.cjs`(新)+ `build/icon.png`(新)+ `build/icon.ico`(新)+ `electron-builder.yml` + `package.json`

### #12 · xterm 搜索崩 React 错误边界

- **现象**:Ctrl+F 弹搜索栏 → 输入文字 → `Uncaught Error: You must set the allowProposedApi option to true to use proposed API`
- **根因**:`TerminalView.tsx:417` 写 `allowProposedApi: false`。`@xterm/addon-search` 内部用 `term.registerDecoration` 高亮命中,这是 xterm 标的 proposed API
- **修复**:`allowProposedApi: true`。注释说明:该 API 在 xterm 5.x 长期稳定使用,proposed 标签只是 RFC 流程慢
- **文件**:`src/renderer/components/TerminalView.tsx`

### #13 · 导入导出按钮宽度不一致

- **现象**:数据分类页 "导出…" 和 "导入…" 按钮一长一短
- **修复**:`.settings-button` 加 `min-width: 96px` + `text-align: center`,所有设置按钮统一规格
- **文件**:`src/renderer/styles/global.css`

### #16 · Explorer 右键 "unable to find electron app"

- **现象**:用户从 alpha.1 升 alpha.2 后,Explorer 右键点 "在 Marina 终端中打开" → Windows 弹错 `Error launching app: unable to find electron app at ...`(指向旧路径)
- **根因**:alpha.1 启用 Explorer 集成后写入 `HKCU\...\Marina\command` = `"C:\旧路径\Marina.exe" --open-here "%1"`。卸载 alpha.1 时**没清注册表**,装 alpha.2 到不同路径,旧 command 仍生效
- **修复**:
  - `build/installer.nsh` 加 NSIS `customUnInstall` macro,卸载时 `DeleteRegKey HKCU "Software\Classes\Directory\shell\Marina"` 与 Background 同名 key,顺手清 EasyTerm 旧 key
  - `electron-builder.yml` nsis 段 `include: build/installer.nsh`
  - 启动期 re-register(早在 alpha.1 就有)继续保留,作为升级用户的自愈路径
- **文件**:`build/installer.nsh`(新)+ `electron-builder.yml`

### #17 · 不可达路径无 toast

- **现象**:从 MainPane 加号按钮或模板按钮新建 session 失败时,只 `console.error`,UI 上无反馈,加号恢复可点
- **修复**:`MainPane.tsx` 三处 catch 块(`EmptyPathState.handleCreate` + `TabBar.handleNewTab` + `MainPane.handleDrop`)统一 `toast.push({ kind: 'error', ... })`
- **文件**:`src/renderer/components/MainPane.tsx`

### #18 · 中文路径 spawn 失败 + 用户看不到原因

- **现象**:在 `C:\Users\liyue\Desktop\新建文件夹` 起 session → `PtySpawnFailed: Cannot create process, error code: 267`,UI 无反馈
- **根因**:Windows API ERROR_DIRECTORY = 267 表示 `lpCurrentDirectory` 无效。可能是:目录被 Explorer 创建后又删了 / 编码异常 / 权限。ConPTY 把 Win32 错误透传,Marina 包成 `PtySpawnFailed` 但用户看不到
- **修复**:
  - `session-manager.createSession` 在 spawn **之前**做 `existsSync(cwd) && statSync(cwd).isDirectory()` 预校验,失败抛 `CwdNotAccessible` 配友好消息(`工作目录 "..." 不存在或不是目录。可能它被删除 / 改名,或路径里含 Marina 无法访问的字符。`)
  - 加 `SessionManagerOptions.skipCwdValidation` 给单测用(伪路径 `'C:\\proj\\a'` 跳过校验)
  - toast 路径已在 #17 修通,用户能看到详细 message
- **文件**:`src/main/session-manager.ts` + `src/main/session-manager.test.ts`
