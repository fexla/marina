## invariant(本地路径零回归)

**核心保证**:无 active remote profile 时,行为 = 今天。

- preload `ensureTransport`:get-active-connection 返回 null → `remoteTransport=null`,
  `invoke/on` 走原 ipcRenderer 逻辑(信封不变)
- snapshot 多两个字段(`remoteBackendProfiles: []`, `activeRemoteProfileId: null`),
  renderer 默认 state 一致,不影响现有 UI
- daemon 模式(`--headless --daemon`)是**可选**启动路径,普通启动不触发
- RemoteProfileManager 是**可选**deps,不注入时 IPC handler 返回空列表/null

---

## TLS 决策(阶段2 简化)

**阶段2 用纯 token 认证,不做 TLS。** 理由:
1. 场景是 WireGuard 内网(决策:LAN direct + WireGuard),WireGuard 本身已加密
2. TLS 自签证书生成需新依赖(node 无原生证书生成,要 selfsigned/node-forge)= 边界2
3. token 认证已防 WireGuard 网内其他设备误连

**风险 R4(TLS 自签)** 降级:WireGuard 内网下中间人风险低。留 V2 / 跨网络场景评估。

## 手动 loopback 测试指南(待开发者执行)

### 准备
1. `npm run build`(或 dev)产 Marion.exe
2. 同一台 Windows 机器

### 测试 1:daemon 启动 + token 持久化(预计 2 分钟)
1. 命令行跑 `Marina.exe --headless --daemon --port=12580`
2. **预期**:无窗口启动,托盘图标出现
3. 看 `%APPDATA%/Marina/logs/main.log`,应有:
   - `WS server listening on port 12580`
   - `auth token (new, for pairing): <UUID>`(首次)
4. 右键托盘 → 菜单应有"**复制 daemon 配对 token**"项,点击复制 token
5. **二次启动**(关掉再跑同样命令):日志**不应**再打 token(非 isNew),但托盘菜单仍有"复制 token"
6. **预期**:token 跨重启稳定(同一 token)

### 测试 2:client 配对 + 连接(预计 3 分钟)
1. daemon 持续运行(测试 1 的实例)
2. 另开一个 Marina(正常启动 `Marina.exe`,非 daemon)
3. 设置 → "**远程后端**" 分类(左侧导航,data 后)
4. 点"添加",填:
   - 显示名:`本机 daemon`
   - host:`127.0.0.1`
   - port:`12580`
   - token:粘贴测试 1 复制的 token
5. **预期**:profile 出现在列表,标记"已配对"
6. 点"**连接**"
7. **预期**:标记变"活跃中";窗口重新初始化(短暂闪);之后命令走远程 daemon
8. 在终端跑 `dir`(或任意命令) → **预期**:输出来自 daemon(本机 daemon = 同机,输出一致)
9. 点"**切回本地**" → **预期**:回到本地模式

### 测试 3:错误 token 被拒(预计 1 分钟)
1. 测试 2 基础上,添加第二个 profile,token 故意填错
2. 点"连接" → **预期**:握手失败(控制台 error),profile 不变活跃,本地模式不受影响

### 测试 4:invariant 本地零回归(预计 1 分钟)
1. 不配置任何 remote profile(或全删)
2. 正常用 Marina:开终端、跑命令、改设置、加收藏
3. **预期**:行为与未引入远程后端时完全一致

### 失败时
- daemon 启动失败 → `%APPDATA%/Marina/logs/main.log` 看 `remote-daemon` 开头错误
- client 连不上 → renderer 控制台(DeterminsticCtrl+Shift+I)看 `[preload] remote transport init failed`
- token 不匹配 → daemon 日志 `auth failed: token mismatch`
