# 0.3.2 性能诊断方案：常驻飞行记录器 + 按需 CPU Profile

**状态**：已实现，待开发者运行验收  
**目标版本**：0.3.2（开发阶段保持 package.json 0.3.1，记入 CHANGELOG `[Unreleased]`）  
**日期**：2026-07-22

## 1. 背景

0.3.1 修复了 Git status poller 在 exited session / 关闭窗口后仍持续运行的问题。该问题说明 Marina 缺少可长期留存、能把“系统变卡”映射到 main event loop、Electron 子进程、业务操作与资源数量的现场证据。

0.3.2 增加内置性能诊断。目标不是把 Marina 变成通用 APM，而是让每次运行都自动留下低开销、隐私安全、能支持下一轮根因分析的飞行记录。

## 2. 两层模型

### L1：每次运行自动开启的低开销飞行记录器

- main event-loop delay / utilization；
- timer drift 卡顿次数（>=100ms / >=250ms / >=1000ms）；
- main CPU、RSS/heap/external/arrayBuffers；
- Electron Browser/Tab/GPU/Utility 进程 CPU 与内存汇总；
- active resource 类型计数；
- window / live-exited-total session / Git watcher / in-flight poll gauges；
- 固定名称业务操作热力图（IPC command、git status/diff 等）；
- 平时每 5 分钟原子刷新 JSON + Markdown；>=1 秒严重 stall 至多每 60 秒额外刷新；
- 最近 30 次运行保留；异常退出仍保留 `finalized:false` 的最近快照；
- event-loop histogram 用 100ms resolution（约 10 次低开销 wakeup/s），业务 stall timer 为 250ms。

### L2：用户显式触发的深度 V8 CPU Profile

- 设置 -> 高级中按钮触发；
- 调用前明确提示 `.cpuprofile` 会包含函数名和本地源码路径；
- 默认 15 秒，服务端硬限制 5-30 秒；
- 同时只允许一份，每次运行最多保留最近 5 份；
- 从不因检测到 stall 自动启动，避免 profiler 自己改变现场。

## 3. 明确不做

- 不自动采集调用栈、路径、命令参数、终端内容、环境变量或 IPC payload；
- 不做 renderer dashboard / trace viewer / 火焰图 UI；
- 不把 main event-loop stall 冒充为 renderer frame jank 或 Windows DPC/ISR；
- 不引入第三方 APM/可视化依赖；
- 不把采样间隔做成复杂设置矩阵；
- 不在本期做同仓库多 session Git poller 合并，先让报告给出证据。

## 4. 数据边界

自动报告只能包含：

- 固定枚举操作名；
- 数值指标、计数、桶分布；
- Electron 进程类型和 PID；
- 应用/运行时版本、操作系统版本；
- 相对本次启动的时间。

自动报告禁止包含：

- sessionId / windowId；
- 仓库、文件、用户目录绝对路径；
- shell 命令、PTY 字节、输入内容；
- IPC payload；
- stack trace / function source URL。

`.cpuprofile` 属于敏感诊断物，只能显式触发并留在本机。

## 5. 文件与生命周期

目录：`%APPDATA%/Marina/performance-reports/`

```text
run-20260722T123456Z-<short-id>.json
run-20260722T123456Z-<short-id>.md
run-20260722T123456Z-<short-id>-cpu-20260722T124500Z.cpuprofile
```

启动后立即写 skeleton；10 秒采样；250ms stall timer；5 分钟周期写盘；>=1 秒 stall 按 60 秒限频额外落盘；正常退出在 1 秒 flush budget 内显式等待 final。报告状态有 `finalized` 字段，异常退出时保持 false。

## 6. 关键实现

- `src/main/performance-metrics.ts`：Electron-free、有界 operation/counter/gauge registry；
- `src/main/performance-diagnostics.ts`：采样、stall、报告、retention、Inspector；
- `src/shared/performance-types.ts`：IPC/报告共享 schema；
- `src/main/ipc.ts`：统一 IPC duration 埋点 + 性能命令；
- `src/main/git-service.ts`：git operation、poll skip、watcher/in-flight gauges；
- `src/main/index.ts`：在 app ready 后启动、退出时 flush；
- `src/renderer/components/SettingsView.tsx`：打开目录、捕获 profile。

## 7. 验收

- 每次 app run 立即产生 JSON/Markdown；
- 无窗口 tray-only 时继续采样；
- 报告能看到 stall 次数、event-loop 分位数、CPU/内存峰值、Electron 进程汇总、operation heatmap；
- 自动报告通过隐私字段守护测试；
- CPU profile 显式确认、15 秒、并发拒绝、finally 清理；
- 所有集合有硬上限；
- typecheck/test/lint/lint:css/build 全过。
