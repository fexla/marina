## 0. ⚠️ **第一次启动前必读** — 数据目录迁移

Electron 的 `app.getPath('userData')` 是由 `productName` 派生的:
- 旧 `productName: 'EasyTerm'` → 数据在 `%APPDATA%\EasyTerm\`
- 新 `productName: 'Marina'` → 数据在 `%APPDATA%\Marina\`

改名后启动新版,你会发现:
- 收藏 / 最近 / 模板 / 设置 / 窗口位置 **全部为空**(并不是丢了 — 它们还在旧目录里)
- 任何首次启动行为(默认主题、默认 windowStyle='windows')都会被应用

**手动迁移**(PowerShell):
```powershell
# 把旧目录全部拷到新目录(如果新目录还不存在,启动一次让 Electron 创建后再覆盖,
# 或者用 -Force 直接覆盖默认值)
$old = "$env:APPDATA\EasyTerm"
$new = "$env:APPDATA\Marina"

if (Test-Path $old) {
    if (-not (Test-Path $new)) { New-Item -ItemType Directory -Path $new | Out-Null }
    Copy-Item -Path "$old\*" -Destination $new -Recurse -Force
    Write-Host "已把 $old 全部内容拷到 $new"
    Write-Host "确认新版数据正常后,可以删除旧目录: Remove-Item -Recurse $old"
} else {
    Write-Host "$old 不存在 — 你之前可能没用过 EasyTerm 或已经在别处迁过了。直接启动新版即可。"
}
```

**做完再启动 Marina**。验证:打开应用 → 看到原来的所有收藏 / 最近 / 模板 / 设置;窗口位置在上次关闭的位置;主题等用户改动都在。

**归档(`*.json` 导出文件)向后兼容**:旧 EasyTerm 导出的归档(`format: 'easyterm-archive'`)可以直接在 Marina 里 import,我加了双 format 读侧接受。
