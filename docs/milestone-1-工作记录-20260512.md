## 1. 14 项逐条实现

### M1-A · 自绘标题栏 + macOS/Windows 双布局

**新增**:
- `src/renderer/components/WindowChrome.tsx`(135 行):两个变体 (`windows` 右侧 lucide 按钮 / `macos` 左侧 traffic-light),双击拖动区切最大化,订阅 `evt:window:max-state-changed`。
- `src/shared/types.ts` 加 `WindowStyle` 字面量 + `Settings.appearance.windowStyle`。
- `src/main/settings-manager.ts` 默认 `'windows'`,validate 检查枚举值。
- `src/main/window-manager.ts` 改 `frame: false`;新增 `minimizeWindow / toggleMaximizeWindow / isMaximized`;`close` 事件前回写 bounds 到 settings(via `onBeforeClose` 回调)。
- `src/main/index.ts` 加 `Menu.setApplicationMenu(null)` 关掉 Win 下默认 File/Edit/View 菜单条。
- `src/shared/protocol.ts` 加 3 个 cmd 通道 + 1 个 evt 通道 + 对应 payload。
- `src/main/ipc.ts` 注册三个新 handler。
- `src/renderer/App.tsx` 渲染 `<WindowChrome>`,根 div 加 `data-window-style` attribute。

- `src/renderer/styles/global.css` 加 `.app-titlebar` / `.app-titlebar-windows` / `.app-titlebar-macos` / `.titlebar-traffic` 等约 100 行 CSS。

### M1-B · 拖文件夹到终端区

`src/renderer/components/MainPane.tsx` 在 `.main-pane` 接 `onDragOver/Leave/Drop`,拖文件夹直接调 `SESSION_CREATE`,defaultTemplateId 走全局 default;`.main-pane.drag-over` 加半透明 + 中央"松开鼠标 — 在该文件夹打开新终端"提示。

### M1-C · 右键菜单完整化

新增 `src/renderer/components/ContextMenu.tsx`(全局 Provider,支持 `divider` / `danger` 视觉变体),挂到 `App` 根,Sidebar 删掉内嵌的旧 provider。`SessionManager.renameSession` 新增 + `cmd:session:rename` IPC。

- Sidebar 收藏 path:复制路径 / 在 Explorer 中显示 / 重命名(行内输入,Enter 提交、Esc 取消)/ 移除收藏(danger)/ 设默认模板(模板列表展开)
- Sidebar 临时 / 最近 path:复制路径 / Explorer / 加入收藏 / (最近)从最近移除
- Sidebar session item:重命名(行内)/ 复制 cwd / 复制 PID / 显示完整命令(disabled 用作展示)/ 关闭(danger)
- Tab(主区):重命名 / 复制路径 / 复制 cwd / 在 Explorer 中显示 / 关闭

### M1-D · 主进程崩溃兜底 + 持久化日志

新增 `src/main/logger.ts`(180 行):
- 无新依赖,纯 `fs.createWriteStream`
- 写 `%APPDATA%/EasyTerm/logs/main-YYYY-MM-DD.log`,按日 / 5 MB 切,7 天保留
- 尊重 `settings.advanced.logLevel`(INFO / DEBUG 即改即生效)
- `setLogDir` 前的日志先存内存,绑定后批量 flush(避免启动期日志丢)
- 暴露 `_resetForTest()` 给单测

`main/index.ts`:`process.on('uncaughtException')` + `unhandledRejection` 全装 handler,不让进程死。replace `console.warn/error` 为 `logger.*`。

### M1-E · 应用图标

- `tray.ts` 程序生成的 16×16 占位图升级:加入 ">" 提示符 + active 态绿色光点;`'default' | 'active'` 两态切换。

### M1-G · 窗口位置/尺寸记忆

### M1-H · 托盘菜单完整化 + 动态图标

`tray.ts` 全部重写:
- 完整菜单(spec 6.5.3):打开新窗口 / 显示所有窗口(>=1) / 关闭所有窗口(>=1) / "正在运行的会话" 子菜单 / 设置… / 完全退出 EasyTerm

1. **WindowChrome 视觉**:
   - 启动 dev → 看见自绘标题栏(没有 OS 默认条),Rose Pine 默认主题下整片紫色无割裂

   - 三个按钮(min/max/close)工作;按钮 hover 高亮(close hover 变红)
   - 切到设置 → 外观 → "窗口风格" 选 macOS → 立即变成左侧三色 traffic light;hover 显示内部符号

   - 7 套主题切换时标题栏颜色一起跟,无残留白色

2. **拖文件夹**:
   - 从 Explorer 拖一个文件夹到终端区域 → 全屏出现半透明 dropzone + "松开鼠标…"提示
   - 松手 → 在该路径新建终端,切到它

3. **右键菜单**:
   - Sidebar 收藏路径右键 → 5 项菜单都可点,重命名走行内编辑(Enter 提交 / Esc 取消),移除收藏标红
   - Sidebar 临时 / 最近路径右键 → 不同的菜单项集合
   - Sidebar session item 右键 → 6 项菜单都可点;"完整命令"显示为 disabled hint
   - Tab 右键 → 5 项菜单都可点;不是本窗口持有时"重命名"和"关闭"灰显

4. **托盘菜单**:
   - 右键托盘 → 完整菜单(打开新窗口 / 显示所有 / 关闭所有 / 正在运行的会话子菜单 / 设置 / 完全退出)
   - 子菜单显示 session 列表;点击聚焦到对应窗口并选中该 session
   - 启动一个 watch 命令 → 托盘图标右下角出绿点;Ctrl+C 停止后绿点消失

5. **窗口记忆**:拖到某位置 + 缩放 → 关 → 重开 → 还在原位

6. **多行粘贴 + 搜索 + 字号**:复制多行 → 终端右键粘贴 → 弹 confirm;Ctrl+滚轮 调字号(所有窗口同步)

7. **设置导入**:
   - 设置 → 数据 → 导出(若有 env 应弹三选一)→ 导出文件名带 `-public/-with-secrets` 后缀
   - 选另一份归档导入 → **不重启**,所有窗口立即反映新数据

8. **崩溃兜底**:在 DevTools console 里 `throw new Error('test')` → 应用不死;`%APPDATA%/EasyTerm/logs/` 下能看到日志

9. **模板编辑器**:打开任一模板编辑 → env 字段默认 blur 模糊 → hover/focus 立即清晰 → 离开恢复模糊

10. **不可达路径 toast**:把一个收藏路径手动 rename(系统层) → Sidebar 双击该路径 → 应该弹 toast 错误

11. **关闭 OS 默认菜单**:Alt 键不再弹 File / Edit / View 菜单条
