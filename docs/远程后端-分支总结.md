## 一、功能目标

在一台电脑上使用 Marina UI,与另一台电脑上运行的 CLI 程序交互,并提供完整 Marina 体验(路径/收藏夹/模板/文件面板/终端)。

## 二、整体架构

### 每窗口后端模型(ADR-014)

```
[客户端机器]                              [服务端机器]
┌─────────────────────────────┐         ┌─────────────────────┐
│ Marina 进程                  │         │ Marina daemon 进程   │
│                              │         │ (--instance=daemon)  │
│ ┌─────────┐  ┌─────────┐    │         │                     │
│ │窗口1    │  │窗口2    │    │   WS    │ ┌─────────────────┐ │
│ │backend= │  │backend= │◄──┼─────────┼─┤ SessionManager   │ │
│ │null     │  │profileX │    │ ws://   │ │ PathManager      │ │
│ │(本地)   │  │(远程)   │    │ 32780   │ │ PTY 进程         │ │
│ └────┬────┘  └────┬────┘    │         │ └─────────────────┘ │
│      │            │          │         │                     │
│   本地 main    RemoteTransport        │ 本地窗口(可选)      │
│   后端         (WS client)            │                     │
└──────┼────────────┼──────────┘         └─────────────────────┘
       │            │
       │       连接 daemon 的 WS server,
       │       每个 OS 窗口独立连一个后端,
       │       窗口生命周期内 backend 不变
       │
   本机 IPC
```

**核心原则**:窗口是临时的观察工具,daemon 才是"应用"。每个窗口独立连一个后端(本地或远程),互不干扰。

### 控制面 / 数据面分离(test.4 的关键架构修复)

```
┌──────────────────────────────────────────────────────────┐
│ preload invoke(channel, payload)                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌─── 本地控制面 (永远走客户端 ipcRenderer) ───┐       │
│  │ APP_QUIT / WINDOW_* / REMOTE_PROFILE_*     │       │
│  │ SYSTEM_CLIPBOARD_*                          │       │
│  │ → 当前客户端机器的 Electron main            │       │
│  └─────────────────────────────────────────────┘       │
│                                                          │
│  ┌─── backend 数据面 ──────────────────────────┐       │
│  │ SESSION_* / BOOKMARK_* / SETTINGS_* / ...   │       │
│  │ 本地窗口 → ipcRenderer                       │       │
│  │ 远程窗口 → WS transport → daemon             │       │
│  └─────────────────────────────────────────────┘       │
│                                                          │
│  ┌─── 跨域复合命令 ────────────────────────────┐       │
│  │ SESSION_OPEN_IN_NEW_WINDOW:                 │       │
│  │   daemon release + 客户端本地建窗口 + 新窗口 claim│
│  └─────────────────────────────────────────────┘       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 五、关键技术决策

| 决策 | 理由 |
|------|------|
| V1 用 `ws://` 不用 `wss://` | ADR-015:TLS 延期 V2,WireGuard/局域网信任 |
| CSP `connect-src 'self' ws: wss:` | 远程 host 运行时配置,无法预枚举;桌面应用可接受 |
| 每窗口后端(非全局 active) | 原则 4:窗口平等,不能全局绑定 |
| 控制面/数据面分离 | BrowserWindow 生命周期属于客户端,session/path 属于 daemon |
| clientId vs windowId 分离 | envelope 走 clientId;session ownership 走 ownerClientId;窗口实体操作走 windowId |
| 先订阅后 snapshot | 消除 snapshot→subscribe 空档丢失事件 |

## 六、Review 发现并修复的 bug

### test.4 前修复(用户测试驱动)

| Bug | 现象 | 根因 |
|-----|------|------|
| CSP 阻止 WS | 连不上 daemon | prod CSP `connect-src 'self'` 禁 ws:// |
| 静默回退本地 | 连接信息缺失时窗口变本地 | connection=null 未 throw |
| 终端打不开 | session 创建后 TerminalView 不挂载 | snapshot myWindowId 未用 daemon clientId |
| header 失效 | 最小化/最大化/关闭无效 | WINDOW_* 错发 daemon |
| 跨窗口不同步 | 切不过去/左侧缺项 | owner clientId 混用 + snapshot 后才订阅 |
| 移动 session 异常 | 在新窗口打开创建服务端窗口 | SESSION_OPEN_IN_NEW_WINDOW 整体发 daemon |

### test.4 后 review 修复(c44c257)

| Bug | 严重度 | 根因 |
|-----|--------|------|
| 远程剪贴板走 daemon | 重要 | SYSTEM_CLIPBOARD_* 未列入本地控制面 |
| WsServer 重连竞态 | 中等 | close handler 未检查 ws 实例同一性 |
| 删除 profile 不检查使用 | 次要 | RemoteProfileInUse 码声明但未用 |

## 七、已知限制(V2 待办)

1. **无 TLS**:V1 用 ws://,依赖网络层(WireGuard/局域网)保证安全
2. **端口扫描 32780-32789**:固定 10 端口,daemon 改用别的需手动配
3. **跨后端无 session 共享**:本地窗口看不到远程 session(设计如此)
4. **probe 完整认证**:端口扫描每个 probe 都完整 auth+register,会产生瞬时 client(架构噪声,未来改无注册 probe)
5. **无 CLIENT_LIST_UPDATED 事件**:WindowInfo 列表不含远程 client,renderer 无法判断 owner 是否在线(当前够用)
